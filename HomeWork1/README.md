# Предсказание цены автомобиля

## Описание задачи

Задача регрессии: предсказать цену автомобиля на основе его технических характеристик.

**Датасет:** синтетический датасет на основе структуры [Automobile Data Set (UCI)](https://archive.ics.uci.edu/dataset/10/automobile), 500 записей.

**Целевая переменная:** `price` (цена автомобиля в у.е.)

**Признаки:**

| Признак | Тип | Описание |
|---------|-----|----------|
| `make` | категориальный | Марка автомобиля (Toyota, BMW, ...) |
| `fuel_type` | категориальный | Тип топлива (gas / diesel) |
| `body_style` | категориальный | Тип кузова (sedan, hatchback, ...) |
| `drive_wheels` | категориальный | Привод (fwd / rwd / awd) |
| `engine_size` | числовой | Объём двигателя (куб. дюймы) |
| `horsepower` | числовой | Мощность (л.с.) — **есть пропуски** |
| `curb_weight` | числовой | Снаряжённая масса (кг) — **есть пропуски** |
| `city_mpg` | числовой | Расход в городе (миль/галлон) — **есть пропуски** |
| `highway_mpg` | числовой | Расход на трассе (миль/галлон) |
| `year` | числовой | Год выпуска |

## Методология

Исследование повторяет структуру семинара и состоит из двух частей.

### Часть 1 — Обработка пропущенных значений

Три подхода (только числовые признаки, модель: RandomForestRegressor, метрика: MAE):

1. **Удаление столбцов с пропусками** — самый простой способ, теряем информацию
2. **Imputation (заполнение средним)** — заполняем пропуски средним значением по столбцу
3. **Imputation + флаг пропуска** — заполняем среднее И добавляем булев признак `_was_missing`

### Часть 2 — Кодирование категориальных признаков

Три подхода (числовые + категориальные признаки):

1. **Исключение категориальных** — используем только числовые признаки
2. **Ordinal Encoding** — присваиваем каждой категории целое число
3. **One-Hot Encoding** — создаём бинарный столбец для каждой категории

## Результаты

| Подход | MAE |
|--------|-----|
| 1. Удаление столбцов с пропусками | **8 866** |
| 2. Imputation (среднее) | 9 326 |
| 3. Imputation + флаг пропуска | 9 231 |
| 4. Без категориальных признаков | 8 866 |
| 5. Ordinal Encoding | 8 866 |
| 6. One-Hot Encoding | 8 866 |

**Вывод:** Удаление столбцов с пропусками показало наилучший MAE — в данном датасете пропущие столбцы при заполнении средним вносят шум, ухудшая предсказание. Категориальные признаки не добавили предсказательной силы сверх числовых.
